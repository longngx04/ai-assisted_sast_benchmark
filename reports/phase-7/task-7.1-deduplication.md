## 2. Implementation Details

### 2.1 Multi-Signal Deduplication Key (Task 7.1)
`Deduplicator.generate_dedup_key()` builds a deterministic 16-character SHA-256 key:
```text
key = SHA-256( vulnerability_type : CWE : file : function : line_bucket : sink : snippet_hash )[:16]
```
- Line numbers are bucketed into ranges of 10 lines to catch offset detections within the same method body.
- String fields are normalized (lowercased/trimmed) to ensure formatting differences do not prevent matching.

### 2.2 Canonical Finding Selection Scoring (Task 7.2)
Within each duplicate cluster, `score_finding()` evaluates candidates by a 4-tier hierarchy:
1. **Validation Status Rank**: `validated` (3) > `unvalidated` (2) > `uncertain` (1) > `rejected` (0).
2. **Confidence Score**: Higher confidence preferred.
3. **Evidence Specificity**: Longer, more detailed evidence strings preferred.
4. **Data Flow Steps**: Detailed multi-step data flow preferred.

Non-canonical duplicates are updated with `"duplicate_of": "<canonical_finding_id>"`.

### 2.3 Production Output Artifacts (Task 7.3 & CLI 8.3)
- Produces canonical findings at `results/<experiment-id>/findings.jsonl`.
- Provides standalone CLI tool [scripts/deduplicate_findings.py](file:///home/longngx04/VinSOC/week%202/AI-assisted_SAST_Bechmark/repo/scripts/deduplicate_findings.py).
