## 1. Objective

The goal of Task 1.1 is to define a stable, unified **Finding Schema** (Python dataclass + JSON Schema 2020-12) shared across all benchmark experiments, normalization routines, and downstream validation modules. This schema guarantees that all vulnerability findings generated by LLMs or SAST tools conform to a strict contract with deterministic field types, validation constraints, and lossless serialization.

### 2.2 Validation Logic (`__post_init__`)
The dataclass performs mandatory runtime assertion during instantiation:
- Type checks for all required strings (`isinstance(val, str)`).
- Nullable string verification (`val is None` or `isinstance(val, str)`).
- Probability boundary validation (`0.0 <= confidence <= 1.0` and `0.0 <= validator_confidence <= 1.0`).
- Line number positivity (`start_line >= 1`, `end_line >= 1`) and non-boolean validation.
- Line range order verification (`end_line >= start_line`).
- Data flow list structural check (`isinstance(data_flow, list)` with all elements `str`).

### 2.3 Serialization API
- `from_dict(payload: dict[str, Any]) -> Finding`: Parses raw JSON dictionary, converts enum strings, and raises [FindingSchemaError](file:///home/longngx04/VinSOC/week%202/AI-assisted_SAST_Bechmark/repo/harness/schemas.py#L25) on any invalid value.
- `to_dict() -> dict[str, Any]`: Serializes the finding into a clean JSON-serializable dictionary with string enum representations.

### 2.4 JSON Schema Specification ([schemas/finding.schema.json](file:///home/longngx04/VinSOC/week%202/AI-assisted_SAST_Bechmark/repo/schemas/finding.schema.json))
Created [schemas/finding.schema.json](file:///home/longngx04/VinSOC/week%202/AI-assisted_SAST_Bechmark/repo/schemas/finding.schema.json) adhering to JSON Schema Draft 2020-12:
- `additionalProperties: false` to enforce exact 27-field structure.
- `required`: All 27 fields are present in the JSON object (nullable fields allow `null` type).
- Enforces strict enum limits for `severity` and `validation_status`.
- Numeric constraints (`minimum: 0.0`, `maximum: 1.0` for probabilities; `minimum: 1` for line numbers).
